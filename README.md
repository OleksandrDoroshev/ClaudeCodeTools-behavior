# Claude Code — наставницький конфіг

Набір правил, скілів і агентів, які перетворюють Claude Code на AI-наставника.

## Що всередині

```
.claude/
├── CLAUDE.md              ← головний файл правил (238 рядків)
├── skills/
│   ├── preflight/          ← аналіз плану перед стартом
│   ├── mentor-review/      ← рев'ю після завершення
│   ├── reflect/            ← ретроспектива сесії
│   ├── compare/            ← порівняння підходів
│   └── explain/            ← пояснення з перших принципів
└── agents/
    └── rubber-duck.md      ← сократівський дебагер
```

## Встановлення

Скопіюй вміст `.claude/` у домашню директорію користувача:

```bash
# Якщо ти вже в директорії transfer:
cp -r .claude/ ~/

# Або повний шлях:
cp -r /шлях/до/transfer/.claude/ ~/
```

Перевір:

```bash
ls ~/.claude/CLAUDE.md          # головний файл
ls ~/.claude/skills/            # 5 скілів
ls ~/.claude/agents/            # 1 агент
```

Готово. Наступна сесія Claude Code підхопить усе автоматично.

## Як це працює

CLAUDE.md завантажується на старті кожної сесії. Він налаштовує Claude як
наставника: аналізує твої запити на неоднозначність, виправляє помилки,
пропонує скіли в потрібний момент.

Скіли викликаються через `/` — наприклад `/preflight` перед значною роботою.
Агенти викликаються через `@rubber-duck` або автоматично за контекстом.

Детальніше про кожен інструмент — у самому CLAUDE.md, секція
"Скіли та агенти — коли пропонувати".
