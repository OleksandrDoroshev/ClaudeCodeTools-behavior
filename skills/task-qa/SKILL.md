---
name: task-qa
description: QA-тестування реалізованої задачі. Запускає незалежного QA-агента і (для medium/large) edge-case тестувальника.
---

Ти допомагаєш перевірити якість реалізації. Твоя мета — знайти проблеми до того, як їх знайде клієнт.

## Крок 1: Пошук задачі

Якщо користувач явно вказав задачу — візьми її. Якщо ні — знайди останню реалізовану задачу:
1. Подивись `.claude/tasks/` — знайди файл з `Status: in_progress` і відповідним `_plan.md`
2. Якщо таких кілька — запитай яку перевіряти
3. Якщо немає — скажи: "Немає задачі для перевірки."

## Крок 2: Завантаження контексту

Прочитай:
1. `.claude/client.md` — контекст клієнта
2. `.claude/tasks/<date>_<slug>.md` — файл задачі
3. `.claude/tasks/<date>_<slug>_plan.md` — план реалізації

## Крок 3: QA-агент

Запусти агента **qa-tester**. Передай йому повні шляхи до файлів:

```
Агент: qa-tester
Задача: Перевір реалізацію задачі.
Файли:
  - Task: .claude/tasks/<date>_<slug>.md
  - Plan: .claude/tasks/<date>_<slug>_plan.md
  - Client: .claude/client.md
```

**Важливо:** qa-tester запускається як окремий Agent (subagent_type не вказуй — він сам визначений через agent definition). Він не бачить історію сесії, тільки передані файли.

Дочекайся результату qa-tester.

## Крок 4: Результати QA

Покажи результати qa-tester користувачу.

Якщо qa-tester знайшов проблеми 🔴:
- Запропонуй виправити їх до ручного тестування
- Після виправлення — повтори QA (Крок 3)

Якщо проблем немає або тільки 🟢:
- Переходь до ручного тестування

## Крок 5: Ручне тестування

Запитай користувача: "Перевірив вручну? Все працює як очікувалося?"

Якщо є проблеми при ручному тестуванні:
- Зафіксуй їх у файлі задачі (секція "Unresolved questions" або "Assumptions")
- Виправ проблеми
- Повернись до Кроку 3 (QA)

## Крок 6: Edge-case тестування (medium/large)

Якщо категорія задачі medium або large:

Запусти агента **edge-case-tester**:
```
Агент: edge-case-tester
Задача: Знайди граничні випадки для цієї реалізації.
Файли:
  - Task: .claude/tasks/<date>_<slug>.md
  - Plan: .claude/tasks/<date>_<slug>_plan.md
```

Покажи результати. Якщо знайдено 🔴 проблеми — виправ і повтори QA.

Для small/tiny задач — пропусти цей крок.

## Крок 7: Оновлення контексту

Запитай: "Чи дізнався щось нове про клієнта або проєкт під час QA?"

Якщо так:
- Нову інфу про клієнта → онови `.claude/client.md`
- Критичні нотатки (наприклад, "валідація в цьому модулі завжди має бути перед записом у БД") → додай у секцію "Critical notes" у `.claude/CLAUDE.md`

Запитай: "Позначити задачу як done?" Якщо так — зміни статус у файлі задачі на `Status: done`.

## Крок 8: Фінал

Виведи підсумок:
- QA-агент: проблем знайдено N (🔴 X, 🟡 Y, 🟢 Z)
- Edge-case тестування: сценаріїв N, проблем M (якщо запускали)
- Ручний тест: пройдено / є зауваження

Запропонуй наступний крок:
- Якщо все ок: "Задача готова. Можна деплоїти — запусти /deploy-check."
- Якщо є дрібні зауваження: "Виправ <конкретні пункти>, потім можна деплоїти."
