---

Ти допомагаєш ініціалізувати новий клієнтський проєкт. Твоя задача — зібрати вичерпну інформацію про клієнта, створити структуру `.claude/` і підготувати проєкт до роботи.

## Крок 1: Перевірка

Перевір, чи вже існує `.claude/client.md` у поточній директорії. Якщо так — скажи що проєкт вже ініціалізовано, покажи client.md і запитай чи потрібно оновити інформацію (перейти до Кроку 3 для оновлення конкретних категорій).

Якщо client.md немає — продовжуй.

## Крок 2: Збір інформації про клієнта

Збирай інформацію в **три етапи**. Для кожної категорії поясни чому це важливо (один короткий рядок). Якщо користувач відповідає "не знаю" — запропонуй конкретні варіанти або обери сам на основі контексту, позначивши `(assumption)`.

### Етап 1: Базова інформація

Запитай:
1. **Назва клієнта** — як називати проєкт? (ім'я, компанія, продукт)
2. **Чисельність та розмір** — маленький бізнес (1-10 співробітників), середній (10-100), великий (100+), enterprise?
3. **Географічне розташування** — де знаходиться клієнт? Це важливо для локалізації, timezone, legal compliance.
4. **Сфера діяльності** — чим займається? (e-commerce, healthcare, fintech, education тощо)

### Етап 2: Бізнес-контекст

Запитай:
5. **Цільова аудиторія клієнта** — хто кінцеві користувачі? Вік, технічний рівень, потреби.
6. **Мета та ідея діяльності** — навіщо цей бізнес/продукт існує? Яка головна цінність?
7. **Відомі проблеми та побажання** — що болить зараз? Що клієнт хоче вирішити?
8. **Фінансова політика та платоспроможність** (опціонально) — бюджетні обмеження, модель оплати, чи є ризики несплати? Якщо не хочеш відповідати — пропусти.

### Етап 3: Технічний контекст

Запитай:
9. **Щоденний флоу та інструменти** — які сервіси/інструменти використовує клієнт щодня? (CRM, ERP, пошта, месенджери, специфічні платформи)
10. **Доступи та репозиторії** — які є репо (GitHub/GitLab/Bitbucket URLs), доступ до хостингу, API ключі (не передавай секрети в чат, просто познач що вони є), доступ до серверів.
11. **Технічні обмеження** — чи є вимоги до стеку, хостингу, legacy системи з якими треба інтегруватися?

## Крок 3: Створення файлів

Створи структуру `.claude/`:

```
.claude/
├── CLAUDE.md
├── client.md
└── tasks/
```

### client.md

Збери всю зібрану інформацію в `.claude/client.md`:

```markdown
# Client: <назва>

## Базова інформація
- **Назва:** ...
- **Розмір:** ...
- **Географія:** ...
- **Сфера:** ...

## Бізнес-контекст
- **Цільова аудиторія:** ...
- **Мета:** ...
- **Проблеми та побажання:** ...
- **Фінанси:** ... (або "Не зазначено")

## Технічний контекст
- **Інструменти та сервіси:** ...
- **Доступи та репо:** ...
- **Технічні обмеження:** ...

## Історія оновлень
- 2026-05-14: Первинна ініціалізація
```

### CLAUDE.md

Створи `.claude/CLAUDE.md`:

```markdown
# Project: <назва клієнта>

## Instructions
Перш ніж відповідати на будь-який запит, прочитай `.claude/client.md` — там контекст клієнта, його сфера, аудиторія, проблеми та технічні обмеження. Всі рішення мають враховувати цей контекст.

Якщо в `.claude/tasks/` є незавершені задачі (status: in_progress) — нагадай про них на початку сесії.

## Project-specific rules
(правила, специфічні для цього проєкту — накопичуються з кожної таски)

## Critical notes
(критичні проблеми, повторювані помилки, важливі нюанси — накопичуються)
```

## Крок 4: ./init

Запитай користувача: "Яка команда для ініціалізації проєкту? Наприклад: `git clone <url> && cd <dir> && npm install`"

Коли користувач надасть команду:
1. **Покажи** команду що буде виконана
2. **Запитай підтвердження** перед виконанням
3. Якщо команда містить деструктивні операції (`rm -rf`, `git reset --hard`, `sudo`) — попередь і запитай окреме підтвердження
4. Виконай команду
5. Якщо команда впала — покажи помилку, не продовжуй мовчки

## Крок 5: Первинний аналіз кодової бази

Якщо репо успішно склоновано і залежності встановлено:
1. Запусти Explore-агента для аналізу структури проєкту
2. Зафіксуй ключову інформацію в секції "Project-specific rules" у CLAUDE.md:
   - Стек технологій
   - Структура директорій
   - Ключові модулі
   - Як запускати проєкт
   - Як запускати тести

## Крок 6: Фінал

Виведи підсумок:
- Що створено (.claude/CLAUDE.md, .claude/client.md, .claude/tasks/)
- Що клоновано/встановлено
- Ключові факти про клієнта (3-4 пункти)
- Наступний крок: "Коли буде готова задача — просто опиши її, і я запущу /task-start"
