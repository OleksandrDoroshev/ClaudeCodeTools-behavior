Ти допомагаєш створити план реалізації для задачі. Твоя мета — продумати підхід до того, як писати код.

## Крок 1: Пошук задачі

Якщо користувач явно вказав задачу — візьми її. Якщо ні — знайди останню незаплановану задачу:
1. Подивись `.claude/tasks/` — знайди файл з `Status: in_progress` і без відповідного `_plan.md`
2. Якщо таких кілька — запитай яку планувати
3. Якщо немає жодної — скажи: "Немає активної задачі. Запусти /task-start спочатку."

## Крок 2: Завантаження контексту

Прочитай:
1. `.claude/client.md` — контекст клієнта
2. `.claude/CLAUDE.md` — правила проєкту та критичні нотатки
3. Вибраний файл задачі `.claude/tasks/<date>_<slug>.md`

## Крок 3: Дослідження кодової бази

Запусти Explore-агентів (1-3 шт) для дослідження релевантних частин коду:

- **Де знаходиться код, який треба змінити?** (пошук файлів, модулів, компонентів)
- **Які є залежності?** (імпорти, API, база даних)
- **Які є приклади схожих рішень у коді?** (патерни які вже використовуються)
- **Які тести існують?** (щоб знати що не зламати)

## Крок 4: Планування

На основі зібраної інформації, увійди в режим планування (EnterPlanMode) і створи план.

План має містити:
1. **Контекст** — що робимо і чому
2. **Підхід** — як саме реалізуємо
3. **Файли для зміни** — конкретний список із шляхами
4. **Кроки** — послідовність дій
5. **Ризики** — що може піти не так
6. **Верифікація** — як перевірити що все працює

У план-моді план буде записано в системний план-файл (`~/.claude/plans/...`).

## Крок 4.1: Збереження плану в задачу

**Одразу після ExitPlanMode**, коли користувач підтвердив план:
1. Прочитай системний план-файл (той що був створений у план-моді)
2. Запиши його вміст у `.claude/tasks/<date>_<slug>_plan.md`
3. Не змінюй вміст — просто скопіюй як є

## Крок 5: Клієнтський контекст

Перед фіналізацією плану перевір:
- Чи враховано цільову аудиторію клієнта? (наприклад, "пенсіонери" → UI має бути максимально простим)
- Чи враховано відомі проблеми клієнта?
- Чи не конфліктує рішення з щоденним флоу користувачів?
- Чи враховано критичні нотатки з CLAUDE.md?

Якщо під час планування дізнався щось нове про клієнта або проєкт:
- Запитай користувача чи додати це в client.md
- Критичні нотатки одразу додавай у секцію "Critical notes" у CLAUDE.md

## Крок 6: Фінал

Виведи підсумок:
- План збережено: `.claude/tasks/<date>_<slug>_plan.md`
- Ключові файли для зміни: N
- Кількість кроків: N

Запропонуй: "Хочеш запустити /preflight для перевірки плану перед реалізацією?"
